Everytime you run YUM, you get the following message:
Warning: RPMDB altered outside of yum.
Ce message vous informe qu’au moins un paquet à été installé à l’aide de RPM; sans passer par YUM. En conséquence, les différents caches utilisés par YUM sont potentiellement désynchronisés de la base de donnée de RPM (“RPMDB”).

Pour faire disparaître cet avertissement, il suffit de purger les cache de YUM afin de forcer celui-ci à les reconstruire à la prochaine exécution. Pour cela:
[text toolbar="false"]yum clean all[/text]

Afin de prévenir l'apparition de ce message, il suffit d’utiliser yum et non rpm pour installation de paquet.
Concrètement, on utilisera:
[text toolbar="false"]yum --nogpgcheck install package-4.6.3.rpm[/text]
en remplacement de
[text toolbar="false"]rpm --install --test package-4.6.3.rpm[/text]

Sachez qu’une commande localinstall existe pour YUM. Elle se comporte comme install suivi d’une nom de fichier .rpm. Bien qu’elle soit maintenue pour des raisons de compatibilité, son utilisation est déconseillée dans la page de man de yum.
